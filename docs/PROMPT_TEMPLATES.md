### **API 開發 Prompt**
```
## RESTful API 開發

基於之前設計的資料庫結構，請為我的 [專案名稱] 開發完整的 API：

### API 需求列表

**認證相關**:
- POST /auth/register (用戶註冊)
- POST /auth/login (用戶登入)
- GET /auth/profile (獲取用戶資訊)
- PUT /auth/profile (更新用戶資訊)
- POST /auth/logout (用戶登出)

**[業務功能模組]**:
- GET /api/[資源] (列表查詢)
- GET /api/[資源]/:id (單筆查詢)
- POST /api/[資源] (新增)
- PUT /api/[資源]/:id (更新)
- DELETE /api/[資源]/:id (刪除)

### 技術要求
- 使用 Express.js 框架
- JWT 身份驗證
- 輸入驗證 (joi/yup)
- 統一錯誤處理
- API 文檔 (Swagger)
- 日誌記錄
- 速率限制

### 每個端點請包含:
1. 🛣️ **路由處理函數** (完整程式碼)
2. ✅ **輸入驗證** (參數驗證邏輯)
3. 🔐 **權限檢查** (角色權限控制)
4. 🚫 **錯誤處理** (各種異常情況)
5. 📝 **API 文檔** (Swagger 註解)
6. 🧪 **測試案例** (Jest 測試)
```

## 💡 **使用技巧**

### **Prompt 最佳實踐**

1. **具體明確**: 提供詳細的需求和限制條件
2. **結構化**: 使用清楚的格式和分段
3. **包含背景**: 說明專案背景和業務需求
4. **明確產出**: 清楚說明期望的輸出格式
5. **迭代改進**: 根據回應調整和深化需求

### **分階段提問策略**

```bash
# 階段 1: 需求分析
使用需求分析 Prompt → 確定技術方向

# 階段 2: 專案初始化  
mursfoto create project-name --template [選定模板]

# 階段 3: 詳細設計
使用資料庫設計 Prompt → 完成資料架構
使用 API 開發 Prompt → 完成後端開發
使用前端開發 Prompt → 完成前端實現

# 階段 4: 測試與部署
使用測試策略 Prompt → 完成品質保證
使用部署配置 Prompt → 完成系統上線
```
